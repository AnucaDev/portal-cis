🎯 Introducción al Proyecto

El Centro de Intervención Psicoeducativa (CIS) es una institución especializada en la atención integral de niños con dificultades del aprendizaje y trastornos del neurodesarrollo. Nuestro enfoque multidisciplinario combina tratamientos especializados con la participación activa de las familias.

📍 Misión

Acompañar el desarrollo integral de cada niño y familia, ofreciendo intervenciones especializadas en dificultades del aprendizaje, el lenguaje y el neurodesarrollo, desde un enfoque respetuoso, empático y basado en la evidencia científica.

📱 Diseño Responsivo

  • Mobile-first: Diseño optimizado para móviles primero
  • Breakpoints: 400px (móvil pequeño), 768px (tablet), 850px (portal), 1024px (desktop)
  • Sin animaciones: Eliminadas por decisión de diseño — interfaz limpia y directa
  • Grid system: CSS Grid y Flexbox nativos

🔒 Control de Acceso (Frontend)

  • Formulario de cita: Solo visible si hay sesión activa en sessionStorage. Sin sesión → aviso con botones de login/registro
  • Portal familias: Guardia en portal-app.js — redirige a login si no hay cis_usuario
  • Portal profesional: Guardia en profesional-app.js — redirige a login si no hay cis_pro
  • Logout: Limpia sessionStorage antes de redirigir

🔐 Seguridad implementada

  • Contraseñas: Hasheadas con bcryptjs (salt rounds: 12)
  • SQL Injection: Uso de prepared statements con mysql2 parametrizado
  • XSS: Función esc() en renderizado dinámico del portal
  • CORS: Orígenes permitidos: localhost, 127.0.0.1, y file:// para desarrollo
  • Mensajes genéricos en login: No se revela si el email existe o no
  • Código de acceso para profesionales: Configurable en .env (CODIGO_PROFESIONAL)
  • Validación de rol: Cada endpoint comprueba el rol del usuario antes de operar
  • Tokens de consentimiento: 64 caracteres hex generados con crypto.randomBytes(32)

👤 Tabla usuarios

CREATE TABLE usuarios (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nombre VARCHAR(100) NOT NULL,
  email VARCHAR(100) NOT NULL UNIQUE,
  telefono VARCHAR(20),
  password_hash VARCHAR(255) NOT NULL, -- bcrypt, salt 12
  rol ENUM('familia','profesional','admin') DEFAULT 'familia',
  activo TINYINT(1) DEFAULT 1,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

👶 Tabla pacientes

CREATE TABLE pacientes (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nombre VARCHAR(100) NOT NULL,
  fecha_nacimiento DATE,
  genero ENUM('masculino','femenino','otro'),
  diagnostico_principal VARCHAR(200),
  observaciones TEXT,
  profesional_id INT NOT NULL, -- FK → usuarios (rol=profesional)
  tutor_usuario_id INT, -- FK → usuarios (rol=familia), NULL hasta aceptar
  activo TINYINT(1) DEFAULT 1,
  created_at / updated_at DATETIME
);

✅ Tabla consentimientos

CREATE TABLE consentimientos (
  id INT AUTO_INCREMENT PRIMARY KEY,
  paciente_id INT NOT NULL, -- FK → pacientes
  email_tutor VARCHAR(100), -- email del tutor legal
  nombre_tutor VARCHAR(100),
  token VARCHAR(64) UNIQUE, -- 64 hex chars (crypto.randomBytes)
  estado ENUM('pendiente','aceptado','rechazado') DEFAULT 'pendiente',
  fecha_envio DATETIME DEFAULT CURRENT_TIMESTAMP,
  fecha_respuesta DATETIME
);

🔄 Flujos de Usuario

Flujo 1: Familia — Registro y primera cita

1. Visita index.html → sección #servicios
2. Sin sesión → ve aviso "Acceso reservado a familias registradas"
3. Pulsa "Crear cuenta gratis" → registro.html
4. Rellena formulario → fetch POST /api/registro
5. Servidor crea usuario, hashea contraseña, devuelve datos
6. Frontend guarda sesión en sessionStorage (cis_usuario)
7. Redirige a portal.html → historial vacío (usuario nuevo)
8. Vuelve a index.html → formulario de cita ya relleno con sus datos

Flujo 2: Profesional — Crear perfil de paciente

1. Va a login-profesional.html
2. Si no tiene cuenta → registro-profesional.html + código CIS-PRO-2026
3. fetch POST /api/registro-profesional → valida código
4. Sesión guardada en sessionStorage (cis_pro)
5. Redirige a profesional-portal.html
6. Rellena formulario: datos del niño + email del tutor legal
7. fetch POST /api/pacientes → crea paciente + token de consentimiento
8. Token guardado en tabla consentimientos (estado: pendiente)

Flujo 3: Familia — Aceptar consentimiento

1. Tutor legal entra en portal.html con su cuenta
2. fetch GET /api/consentimientos/pendientes → busca por su email
3. Ve tarjeta: "El profesional X solicita vincular a [Nombre niño]"
4. Pulsa "Aceptar" → fetch POST /api/consentimientos/:token/responder
5. Servidor: actualiza estado a 'aceptado' + vincula tutor_usuario_id en pacientes
6. El niño aparece en "Mis pacientes vinculados"

Flujo 4: Administradora — Acceso al panel de gestión

1. Va a login-admin.html
2. Introduce email y contraseña de la cuenta con rol='admin'
3. fetch POST /api/login-admin → verifica rol='admin'
4. Sesión guardada en sessionStorage (cis_admin)
5. Redirige a admin-portal.html
6. Carga automáticamente el Dashboard con 8 estadísticas globales
7. Navega entre: Usuarios • Pacientes • Citas • Consentimientos

2. Doble clic en: api/instalar_dependencias.bat
→ Instala: express, mysql2, bcryptjs, cors, dotenv

3. Doble clic en: api/ejecutar_api.bat
→ Servidor corriendo en http://localhost:3001

4. Verificar: abrir http://localhost:3001/api/health en el navegador
→ Respuesta: { "ok": true, "bd": "conectada" }
